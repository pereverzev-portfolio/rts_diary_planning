пользователь хочет использовать приложение посредством просмотра малого поднабора задач из большого набора задач чтобы планировать выполнение задач в условии нехватки времени и обходить ситуацию, в которой задачи лежат подолгу нвыполненными, забываются

- нужно понимать, какие задачи 
    - вот-вот окажутся просроченными (+если не начать их делать сейчас, с учётом времени на выполнение), 
    - какие уже оказались просроченными, 
    - какие ещё не просрочены
    - какие задачи входят в поднабор, который показывается пользователю
- нужно понимать 
    - описание задачи (текст)
    - время на выполнение задачи (сложность), 
    - время возникновения задачи (в реальной жизни, не время ввода задачи в программу), 
    - дедлайн (его может и не быть),
    - важность задачи,
    - приступали ли к её выполнению, сколько выполнено
- так как можно отметить, что выполнена доля задач, можно сделать историю отметок долей выполнения и возможность редактировать ошибки в истории. Можно отметить факт выполнения:
    - в доле выполненной задачи (только) — тогда будет посчитано количество затраченных часов и оставшееся количество часов на выполнение задачи
    - в часах работы (только) — покажется оставшееся количество часов
    - в доле и в часах одновременно — посчитается доля от предшествующего оставшегося времени, вычтется из затраченного в "подход выполнения", разность прибавится к общему количеству часов на выполнение задачи
- нужно учитывать "график деятельности" пользователя (люди спят, задачи могут гореть перед тем, как человек уснёт, а не когда он уже будет спать)


таким образом, должны быть "места":
- подвыборка задач для выбора на выполнение
- создание новой задачи, выставление ей важности, трудоёмкости, времени возникновения, дедлайна
- показа всех задач
- показа задач, которые можно успеть выполнить вовремя, но они вот-вот будут просрочены
- показа только просроченных задач
___
- отметки о выполнении части задачи
- смены истории отметок о выполнении части задачи
- смены приоритетности задач


Для простоты проекта на портфолио: 
- приложение используется только запущенным
- только Android

При открытии приложения видно "место" подвыборки всплывших по изменённому рпиоритету задач
При открытии приложения можно в минимальное количество кликов создать новую задачу и настроить её жизнь
При желании можно просмотреть место просроченных задач, место всех задач, место подходящих к просрочке задач, историю отметок о выполнении задач

Приоритет задачи составляется из важности задачи и изменения в ходе истории жизни задачи.
Важность задачи имеет смысл ранжирования задач (по важности задачи можно однозначно сравнить на "большее-меньшее", но абсолютное положение смысла не имеет)
Можно сделать равноважные задачи (!)
Установку важности можно сделать выставлением порядка (ранжированием) задач. Тем не менее, нужно иметь возможность отразить "пропасть" между некоторыми задачами.

ВРЕМЕННО: пусть разность важности с предыдущей задачей можно задать в доле разности предыдущей и следующей вниз за ней (с отличающейся важностью) задач от 0 до 1 доли их разности, либо в дробном положительном количестве долей (т.е. либо ползунком, либо в окошке ввода числа), либо сказать, что задача равноважна предыдущей

Задачи могут выполняться. Если задача выполнена, её присутствие / отсутсвие в программе не должно влиять на выставление приоритетов для существующих невыполненных задач. Если задача выполнена, она не должна показываться в списках к выполнению (месте подвыборки, месте просроченных задач, месте горящих задач)
Можно оставлять выполненные задачи в списке всех задач ранжируя вниз списка.
Существование выполненной задачи не должно влиять на выставление приоритетов для задач. По факту выполнения задачи должны пересчитаться относительные важности, если они зависят от(-носительно) выполненных задач.

Задачи могут удаляться. Аналогичная логика (только в общем списке задач они исчезают).

Выполненная задача может дополняться. В таком случае либо редактируется история выполнения (изменяется доля выполнения), либо удаляется факт о доле выполнения, либо дополняется сложность задачи.
