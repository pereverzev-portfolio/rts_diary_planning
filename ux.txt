пользователь хочет использовать приложение посредством просмотра малого поднабора задач из большого набора задач чтобы планировать выполнение задач в условии нехватки времени и обходить ситуацию, в которой задачи лежат подолгу нвыполненными

- нужно понимать, какие задачи 
    - вот-вот окажутся просроченными (+если не начать их делать сейчас, с учётом времени на выполнение), 
    - какие уже оказались просроченными, 
    - какие ещё не просрочены
    - какие задачи входят в поднабор, который показывается пользователю
- нужно понимать 
    - описание задачи
    - время на выполнение задачи, 
    - время возникновения задачи, 
    - важность задачи,
    - приступали ли к её выполнению, сколько выполнено
- так как можно отметить, что выполнена доля задач, можно сделать историю отметок долей выполнения и взможность редактировать ошибки в истории

таким образом, должны быть "места":
- подвыборка задач для выбора на выполнение
- создание новой задачи, выставление ей важности, трудоёмкости, времени возникновения
- показа всех задач
- показа задач, которые можно успеть выполнить вовремя, но они вот-вот будут просрочены
- показа только просроченных задач
___
- отметки о выполнении части задачи
- смены истории отметок о выполнении части задачи
- смены приоритетности задач


Для простоты проекта на портфолио: 
- приложение используется только запущенным
- только Android

При открытии приложения видно "место" подвыборки всплывших по изменённому рпиоритету задач
При открытии приложения можно в минимальное количество кликов создать новую задачу и настроить её жизнь
При желании можно просмотреть место просроченных задач, место всех задач, место подходящих к просрочке задач, историю отметок о выполнении задач
